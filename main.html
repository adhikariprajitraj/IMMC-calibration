Motivation

Why Model Calibration?

  • A model is only useful if its predictions match reality within acceptable error.
  • Calibration means tuning parameters so that model outputs align with observed data — it is the bridge between theory and empirical evidence.
  • Without calibration, a model is merely a mathematical hypothesis with no empirical grounding.
IMMC framing. Calibration supports credible recommendations and honest uncertainty statements — two key components of a high-scoring solution.
\[ \hat{\theta} = \underset{\theta \in \Theta}{\arg\min} \; \mathcal{L}(\theta;\, \mathbf{y}_{\text{obs}}) \]
Key insight. Every parameter value reported in an IMMC paper should ideally be calibrated against real data, not simply assumed.
03
Overview

Session Goals

By the end of this session, you should be able to:

  • Formulate calibration as an optimisation or inference problem and identify its components.
  • Choose a loss function appropriate to your context and justify that choice in writing.
  • Diagnose fit quality using residual analysis and hold-out validation sets.
  • Report calibrated parameters with quantified uncertainty and clearly stated limitations.
  • Recognise when a model is unidentifiable and propose strategies to address it.
Motivation. Each goal maps directly to an IMMC evaluation criterion: model justification, stated assumptions, sensitivity analysis, and recommendations.

Core Concepts

Model, Parameters, States, Outputs

Any model can be written as a function mapping inputs to predictions:

\[ y_{\text{pred}} = f(x;\,\theta) \]

Inputs \(x\)

Time, dose, distance, policy choice, demographic variable — what we observe or control.

Parameters \(\theta\)

Unknown constants to be calibrated: rates, coefficients, thresholds, decay constants.

Outputs \(y_{\text{pred}}\)

Predicted quantities compared against measured data \(y_{\text{obs}}\) to assess fit.

Dynamic models. States (e.g., \(S(t),\, I(t),\, R(t)\)) evolve over time, but calibration typically targets parameters (e.g., transmission rate \(\beta\)) rather than states directly.
07
Core Concepts

Verification, Validation, and Calibration

Verification

Did we solve the equations correctly? Checks for coding bugs, numerical errors, implementation mistakes. Mathematical correctness.

Calibration

Choose \(\theta\) so the model's numerical output matches observed data. Data-driven fitting. Parameter correctness.

Validation

Do the equations represent the real system well? Checks whether model structure is appropriate. Scientific correctness.

IMMC tip. State explicitly which of these three activities you performed. Distinguishing "equations are verified" from "model is validated against independent data" demonstrates sophisticated methodological awareness.
08
Core Concepts

Data and Error Structure

Observed data almost never equals the model prediction exactly:

\[ y_{\text{obs}} = f(x;\,\theta^\star) + \varepsilon \]

Three Types of Error

  • Measurement noise. Instrument precision, observer error, transcription mistakes.
  • Process noise. Genuine randomness in the underlying system (weather, biology).
  • Model discrepancy. Systematic bias from missing mechanisms (structural error).

Why This Matters

  • The error type determines the appropriate loss function.
  • Model discrepancy creates a lower bound on fit quality — no \(\theta\) can fully close a structural gap.
  • Ignoring these leads to overconfident parameter estimates.

Deterministic Calibration

Residuals: What the Model Got Wrong

For data points \((x_i, y_i)\), the residual is the signed error at point \(i\):

\[ r_i(\theta) = y_i - f(x_i;\,\theta) \]
  • Residuals are the foundation of least-squares losses — we want them collectively small.
  • Residual plots reveal systematic patterns suggesting model improvements.
  • The variance of residuals informs uncertainty estimates for parameters.
  • A random, unstructured scatter of residuals around zero is the gold standard for a well-specified model.
Practical advice. Always plot residuals. A single residual plot can reveal more about model quality than any aggregate metric.
12
Deterministic Calibration

Loss Functions — Choose and Justify

NameFormulaPropertiesRecommended When
SSE \(\sum_i r_i^2\) Convex, differentiable; amplifies large errors Gaussian noise, no severe outliers
RMSE \(\sqrt{\tfrac{1}{n}\sum_i r_i^2}\) Same units as output; easy to interpret Reporting and cross-dataset comparison
MAE \(\tfrac{1}{n}\sum_i |r_i|\) Robust to outliers; less smooth at zero Heavy-tailed noise; median-like fit
Neg. Log-Lik. \(-\log p(\mathbf{y}|\theta)\) Principled; depends on noise model assumed Statistical modelling, uncertainty quantification
IMMC note. You must explain why your chosen loss matches the context — what noise process, what outlier structure, what decision stakes are involved.
13
Deterministic Calibration

Weighted Least Squares

If point \(i\) has standard deviation \(\sigma_i\), downweight it accordingly:

\[ \mathcal{L}(\theta) = \sum_{i=1}^n \left(\frac{r_i(\theta)}{\sigma_i}\right)^2 \]
  • Some sensors are more reliable — e.g., GPS vs. manual observation.
  • Counts with high variance at large values — Poisson-like behaviour.
  • Heteroscedasticity — variance that increases with the response variable.
  • Prevents one data regime from dominating the fit at the expense of others.
Determining weights. Use domain knowledge, replicate measurements, or estimate variance as a function of the predictor. Document your weighting rationale.
14
Deterministic Calibration

Constraints and Regularisation

Parameter Constraints

  • Physical bounds: \(\theta \geq 0\) for rates, probabilities, masses.
  • Conservation laws: fluxes must sum to zero.
  • Ordering: \(\theta_1 < \theta_2\) (lower bound below upper bound).

Regularisation

  • Ridge (L2): penalises large parameter magnitude.
  • Lasso (L1): encourages sparse parameter selection.
  • Smoothness: penalises rapid parameter variation.
\[ \mathcal{L}(\theta) = \underbrace{\text{SSE}(\theta)}_{\text{data fit}} + \underbrace{\lambda \|\theta\|^2}_{\text{regularisation}} \]
IMMC application. Regularisation prevents overfitting when calibrating models with many parameters on limited data — a common situation in competition settings.
15
Deterministic Calibration

Identifiability: Can the Data Pin Down \(\theta\)?

Failure Modes

  • Non-identifiable: infinitely many \(\theta\) produce equally good fits (flat loss landscape).
  • Correlated parameters: increasing \(\theta_1\) compensated by decreasing \(\theta_2\) — saddle ridge in loss.
  • Weakly identifiable: data constrains a combination \((\theta_1 + \theta_2)\) but not each individually.

Symptoms & Remedies

  • Estimates unstable under small data perturbations.
  • Very wide confidence intervals.
  • Optimiser fails to converge.
  • Remedies: richer data, fewer parameters, prior knowledge, regularisation.
IMMC move. If you suspect non-identifiability, report it explicitly and explain what additional data or assumptions would resolve it.

Optimisation

Calibration as an Optimisation Problem

\[ \hat{\theta} = \underset{\theta \in \Theta}{\arg\min} \; \mathcal{L}(\theta) \]

Where \(\Theta\) encodes constraints (bounds, feasibility). Key design choices:

Optimiser

Grid search (small problems), gradient-based (smooth loss), evolutionary or heuristic (complex landscapes).

Initialisation

Starting guesses \(\theta_0\) matter for non-convex problems. Use domain knowledge, literature values, or multi-start.

Stopping Criterion

Gradient tolerance, loss improvement threshold, or maximum iterations. Balance computation time against precision.

For IMMC. Grid search is transparent and easy to explain. Whichever solver you choose, justify the decision in writing.
18
Optimisation

Gradient Descent

Core Idea

Move in the direction of steepest descent of the loss. The learning rate \(\eta\) controls step size.

\[ \theta_{k+1} = \theta_k - \eta \nabla_\theta \mathcal{L}(\theta_k) \]

Practical Considerations

  • Too large \(\eta\): divergence or oscillation.
  • Too small \(\eta\): very slow convergence.
  • Adaptive methods (Adam): per-parameter learning rates, faster in practice.
  • Convergence guarantee: only for convex loss; may terminate at local minimum otherwise.
Think of the loss surface as hilly terrain — gradient descent always walks downhill, but may end up in a valley that is not the global minimum.

Optimisation

Local Minima and Multi-Start Strategy

  • Many calibration loss surfaces are non-convex — they have multiple valleys, ridges, and saddle points.
  • A gradient-based optimiser will find a local minimum near its starting point, which may not be the global optimum.
  • The "best" solution can differ markedly depending on initialisation.
Multi-start strategy. (1) Sample many initial guesses \(\theta_0^{(1)}, \ldots, \theta_0^{(K)}\) — random or structured. (2) Run the optimiser from each starting point. (3) Keep the solution with the lowest final loss value. (4) Report the sensitivity of results to initialisation.
IMMC transparency. Report how many starting points you tried and what range of final loss values you observed. This documents robustness of the solution.
20
Optimisation

Nonlinear Least Squares: Gauss–Newton / Levenberg–Marquardt

For nonlinear \(f(x;\theta)\), pure gradient descent is often inefficient. The LM algorithm is the standard workhorse:

  1. Linearise the model around current \(\theta_k\): compute the Jacobian \(\mathbf{J} = \partial f / \partial \theta\).
  2. Solve an approximate (linearised) least-squares subproblem for the update step \(\delta\theta\).
  3. Update: \(\theta_{k+1} = \theta_k + \delta\theta\). Reduce damping if the loss improved; increase it otherwise.
  4. Repeat until convergence (gradient norm or step size falls below tolerance).

Gauss–Newton

Fast near the solution; may diverge far from it. No damping. Suitable for well-conditioned problems.

Levenberg–Marquardt

Blends Gauss–Newton (fast) and gradient descent (safe) via adaptive damping. The industry standard for nonlinear least squares.

21
Optimisation

Scaling and Units Matter

  • Parameters with very different magnitudes (e.g., \(\theta_1 \sim 10^{-6}\) and \(\theta_2 \sim 10^3\)) cause ill-conditioning.
  • A loss dominated by large-unit variables will always over-prioritise that scale in optimisation.

Recommended Fixes

  • Nondimensionalise all variables before calibrating.
  • Reparametrise — optimise \(\log\theta\) for positive rates.
  • Weighted losses normalise contributions across different variables.

Consequences of Poor Scaling

  • Optimiser spends iterations correcting for units rather than finding the true parameter.
  • Condition number of the Hessian worsens, slowing convergence.
  • Gradient norms become unreliable as stopping criteria.

Diagnostics & Validation

What Residual Patterns Tell Us

A well-calibrated model has residuals that resemble random noise. Any pattern signals a problem:

PatternDiagnosisRecommended Action
Curved trendWrong functional form — missing nonlinearityAdd polynomial / spline term or revise model structure
Fan / funnel shapeVariance increases with \(x\) (heteroscedasticity)Use weighted least squares or transform the response
Clusters or groupsMissing subgroup variable or regime changeSegment data; add categorical predictor
Isolated spikesOutliers or rare eventsInvestigate causes; use robust loss; document findings
Autocorrelation (time series)Model missing temporal dynamicsAdd lagged terms; use a dynamic model
IMMC move. Propose specific model improvements suggested directly by residual analysis. This demonstrates analytical depth to judges.
24
Diagnostics & Validation

Outliers and Robust Calibration

  • Squared loss over-penalises large errors — a single outlier can distort the entire fit significantly.

MAE Loss

\(\sum|r_i|\) — treats errors proportionally. Less sensitive to outliers than MSE.

Huber Loss

Quadratic near zero, linear for large residuals. Combines MSE efficiency with outlier robustness via tuning parameter \(\delta\).

Trimmed Fit

Remove extreme residuals before fitting. Must be justified — cannot be used to cherry-pick results selectively.

Explanation is essential. Always provide a real-world reason for outliers: sensor glitches, public holidays, data entry errors, or genuine extreme events. Removing data without justification undermines credibility.
25
Diagnostics & Validation

Training, Validation, and Test Sets

To avoid reporting performance that merely reflects memorisation of noise:

  1. Training set. Fit \(\hat{\theta}\) by minimising the loss function on this data.
  2. Validation set. Tune model-level decisions — model structure, hyperparameters, regularisation strength.
  3. Test set. Evaluate the final, locked model on this untouched data. Report this number in your paper.
Critical rule. The test set must never influence any decision made during calibration. Once examined, it is no longer a valid test set.
IMMC note. Even a simple 70/30 train/test split with a clear written explanation demonstrates sound methodology. Judges reward this over fitting to all available data silently.
26
Diagnostics & Validation

Cross-Validation (When Data is Limited)

K-Fold CV Procedure

  1. Partition data into \(K\) folds of equal size.
  2. For each fold \(k\): train on the remaining \(K-1\) folds, evaluate on fold \(k\).
  3. Average the \(K\) error estimates as overall performance.

When to Use

  • Dataset is small — a separate test split is not affordable.
  • Comparing multiple models or hyperparameter settings.
  • Estimating out-of-sample performance reliably.
\[ \text{CV error} = \frac{1}{K}\sum_{k=1}^K \mathcal{L}(\hat{\theta}_{-k};\, D_k) \]
Leave-one-out CV (LOOCV). Special case \(K = n\). Unbiased but expensive. Appropriate when \(n\) is very small (e.g., epidemic data with few reported time points).
27
Diagnostics & Validation

Time Series — Walk-Forward Validation

  • Time-ordered data must not be randomly shuffled — doing so leaks future information into training.
  • Always train on past data and validate on future data, mirroring real deployment conditions.
Walk-Forward Validation. Fit on \([1 \ldots t]\), evaluate on \((t \ldots t+h]\). Slide the window forward. Average errors across all windows.

Why Standard CV Fails Here

  • Random folds permit training on data from after the test point.
  • Artificially inflates metrics — the model appears better than it truly is.

Relevant IMMC Contexts

  • SIR / epidemic models.
  • Economic or traffic forecasting.
  • Any model predicting future observations from a history of past ones.

Uncertainty Quantification

How Sure Are We About \(\hat{\theta}\)?

Even with perfect optimisation, calibrated estimates vary because data is finite and noisy. Reporting a single point estimate is incomplete.

Parameter Uncertainty

Confidence or credible intervals for each element of \(\theta\). How precisely is each parameter determined by the available data?

Prediction Intervals

Intervals for future outcomes \(y_{\text{new}}\). Wider than parameter CIs — includes both parameter uncertainty and observation noise.

Sensitivity Analysis

How much do model outputs change when parameters vary by ±10%? Documents robustness and identifies critical parameters.

IMMC requirement. Papers should include uncertainty quantification. "\(\beta = 0.3 \pm 0.05\) (95% CI)" is substantially stronger than "\(\beta = 0.3\)."
30
Uncertainty Quantification

Bootstrap Confidence Intervals

The bootstrap is a simple, assumption-light method for quantifying parameter uncertainty:

  1. Resample. Draw \(n\) data points with replacement from your observed dataset.
  2. Recalibrate. Run your optimiser on this bootstrap sample to obtain \(\hat{\theta}^{(b)}\).
  3. Repeat. Perform \(B = 500\) to \(2000\) repetitions to build an empirical distribution of \(\hat{\theta}\).
  4. Summarise. Use the 2.5th and 97.5th percentiles as a 95% confidence interval.
Advantages. No normality assumption; works for any estimator; conceptually straightforward to explain. Limitation. Computationally expensive for slow models; may perform poorly with very small datasets.
31
Uncertainty Quantification

Bayesian Calibration

Instead of finding the single best \(\hat{\theta}\), Bayesian calibration finds the distribution of plausible parameters:

\[ \underbrace{p(\theta \mid \text{data})}_{\text{posterior}} \;\propto\; \underbrace{p(\text{data} \mid \theta)}_{\text{likelihood}} \cdot \underbrace{p(\theta)}_{\text{prior}} \]

Prior \(p(\theta)\)

Encodes plausible values before seeing data. Can incorporate literature values, physical constraints, and expert knowledge.

Likelihood \(p(\text{data}|\theta)\)

Probability of observing the data given parameter values. Encodes the assumed noise model.

Posterior \(p(\theta|\text{data})\)

Full uncertainty-aware calibrated distribution. Summarised by mean, mode, or credible intervals.

Sampling. When the posterior has no closed form, Markov Chain Monte Carlo (MCMC) samples from it numerically (e.g., Metropolis–Hastings or NUTS, as used in Stan / PyMC).
32
Uncertainty Quantification

Model Discrepancy (Structural Bias)

Sometimes no choice of \(\theta\) fits well — not because the optimiser failed, but because the model is structurally incomplete:

  • The model is missing a mechanism present in the real system.
  • Data comes from multiple regimes (weekday vs. weekend, pre- vs. post-intervention, seasonal).
  • External factors shift over time (policy change, measurement system change).

Formal Treatment

Add an explicit discrepancy term \(\delta(x)\):

\(y = f(x;\theta) + \delta(x) + \varepsilon\)

Model \(\delta\) as a Gaussian process (Kennedy–O'Hagan framework).

IMMC-Friendly Approaches

  • Segment the data into distinct regimes and calibrate each separately.
  • Propose extensions to the model that could reduce discrepancy.
  • Report honestly that residual patterns suggest structural misspecification.

Probability Calibration

What Does a Calibrated Probability Mean?

When a model outputs a probability \(\hat{p}\) (e.g., "70% chance of rain"), calibration requires:

\[ \mathbb{P}(Y=1 \mid \hat{p} = q) \approx q \quad \text{for all } q \in [0,1] \]

Among all predictions where \(\hat{p} = 0.7\), approximately 70% should actually be positive events.

Brier Score

Mean squared error of probabilities: \(\frac{1}{n}\sum(\hat{p}_i - y_i)^2\). Lower is better.

Log Loss

Penalises confident wrong predictions heavily: \(-\frac{1}{n}\sum y_i \log\hat{p}_i\). A strictly proper scoring rule.

ECE

Expected Calibration Error — average gap between predicted probability and observed frequency across bins.

35
Probability Calibration

Post-Processing Methods

Platt Scaling

Fit a logistic function mapping raw model scores \(s\) to probabilities: \(\hat{p} = \sigma(as + b)\). Two parameters; works well with small validation sets.

Isotonic Regression

Flexible monotone mapping (non-parametric). More powerful than Platt scaling but requires more data to avoid overfitting the calibration step itself.

Temperature Scaling

Common in deep learning: \(\hat{p} = \text{softmax}(\mathbf{z}/T)\). A single parameter \(T > 1\) softens overconfident predictions.

Critical rule. Always calibrate using a separate validation set held out from model training. Calibrating on training data causes information leakage and produces overconfident probabilities.
IMMC relevance. If your model outputs probabilities (e.g., probability of exceeding a threshold), calibration ensures that recommendations are risk-appropriate.

Case Study A

Calibrating a Spring Model (Hooke's Law)

\[ y = k F \]

Where \(F\) is applied force (N) and \(y\) is extension (m). Calibrate compliance \(k\) (m/N).

Procedure

  1. Collect pairs \((F_i, y_i)\) for several force levels.
  2. Fit by minimising \(\sum_i (y_i - kF_i)^2\); closed form: \(\hat{k} = \frac{\sum F_i y_i}{\sum F_i^2}\).
  3. Validate using forces not included in calibration.
  4. Plot residuals \(r_i = y_i - \hat{k}F_i\) and interpret.

Discussion Points

  • Do residuals suggest that a linear model is adequate?
  • What are the limits of Hooke's Law (elastic limit)?
  • How does measurement error in \(F\) affect the estimate of \(k\)? (errors-in-variables problem)
Deliverable. Point estimate of \(k\) with units, a residual plot, and one paragraph stating assumptions and limitations.

Case Study B

Calibrating an SIR Epidemic Model

\[ \frac{dS}{dt} = -\beta SI, \quad \frac{dI}{dt} = \beta SI - \gamma I, \quad \frac{dR}{dt} = \gamma I \]

Calibration Setup

  • Target parameters: \(\beta\) (transmission) and \(\gamma\) (recovery rate).
  • Fit to observed daily case counts (proxy for \(I(t)\)).
  • Minimise \(\sum_t (I_{\text{obs}}(t) - I(t;\beta,\gamma))^2\) via ODE solver plus gradient-free optimiser.

Key Discussion Points

  • Identifiability: can \(\beta\) and \(\gamma\) be separately identified from \(I(t)\) alone?
  • Under-reporting: observed cases \(\neq\) true \(I(t)\). How does this bias \(\hat{\beta}\)?
  • Derived quantity: \(R_0 = \beta/\gamma\) — report with uncertainty propagated via bootstrap.
IMMC sophistication. Discussing under-reporting bias and identifiability explicitly — even without resolving them — significantly strengthens a paper.

Team Exercise

Logistic Growth Calibration

\[ y(t) = \frac{K}{1 + A\, e^{-rt}} \]

Given time-series data \((t_i, y_i)\), calibrate carrying capacity \(K\), initial-condition factor \(A\), and growth rate \(r\).

Tasks

  1. Choose SSE or MAE loss; write a one-paragraph justification.
  2. Estimate \((K, A, r)\) via grid search or a numerical solver.
  3. Validate by holding out the last 20% of time points.
  4. Report uncertainty via bootstrap (≥ 200 replicates) or parameter sensitivity.

Discussion

  • How sensitive is \(\hat{K}\) to the proportion of data available near saturation?
  • What happens if only early-growth data is available? Is \(K\) identifiable?
  • Can you construct a 95% confidence interval for \(K\)?
  • How does the model perform on the held-out set?

Section VIII — Final Reference

IMMC Reporting Checklist

Problem Setup

  • All variables defined with units stated.
  • Calibrated vs. assumed parameters identified.
  • All modelling assumptions stated explicitly.
  • Data source, quality, and limitations described.

Calibration Procedure

  • Loss function explained and justified.
  • Optimisation method and initialisation described.
  • Convergence criteria and final loss value reported.
  • Calibrated parameters reported with uncertainty bounds.

Validation & Diagnostics

  • Residual plots included and patterns interpreted.
  • Out-of-sample / validation performance reported.
  • Identifiability concerns discussed where relevant.

Communication

  • Parameter uncertainty and prediction intervals stated.
  • Sensitivity analysis performed on key parameters.
  • Limitations and model discrepancy clearly stated.
  • Calibration results connected to recommendations.
