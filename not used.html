Model Calibration

Parameter Estimation & Validation Strategy for IMMC

Prajit Adhikari · 2026 Training Series
01 · Intro

Why Model Calibration?

  • Predictions must match reality within acceptable error.
  • Tuning parameters bridges theory and empirical evidence.
  • Without it, a model is just a hypothesis without grounding.
\[ \hat{\theta} = \arg\min_{\theta \in \Theta} \; \mathcal{L}(\theta;\, \mathbf{y}_{\text{obs}}) \]
IMMC solutions require "credible recommendations"—this requires calibration.

Part I

Core Concepts

02 · Concepts

Inputs, Parameters, and Outputs

\[ y_{\text{pred}} = f(x;\,\theta) \]

Inputs ($x$)

Variables we control/observe: Time, dose, or distance.

Parameters ($\theta$)

Unknown constants: Growth rates, thresholds, decay rates.

Outputs ($y$)

Predicted values to be compared with real data.

02 · Concepts

Verification vs. Validation vs. Calibration

Verification

Did we solve the math correctly? (Bug checks).

Calibration

Tuning $\theta$ to match data.

Validation: Does the model actually represent the real world?
02 · Concepts

Data and Error Structure

\[ y_{\text{obs}} = f(x;\,\theta) + \varepsilon \]

Three types of error to account for:

  • Measurement Noise: Instrument precision issues.
  • Process Noise: Randomness in the system (e.g. weather).
  • Model Discrepancy: Systematic bias from missing mechanisms.
02 · Concepts

Calibration Workflow

1. Define Model $\rightarrow$ 2. Choose Loss $\rightarrow$ 3. Optimize $\rightarrow$ 4. Validate $\rightarrow$ 5. Report.

This loop may iterate if validation performance is poor.
Part II

Deterministic Calibration

03 · Deterministic

Residual Analysis

The residual is the "error" at each point $i$:

\[ r_i(\theta) = y_i - f(x_i;\,\theta) \]
  • Residual plots reveal systematic patterns.
  • Ideal residuals are a random scatter around zero.
03 · Deterministic

Choosing a Loss Function

NameFormulaRecommended When
SSE$\sum r_i^2$Gaussian noise, no outliers.
RMSE$\sqrt{\text{SSE}/n}$Easy interpretation (same units).
MAE$\frac{1}{n} \sum |r_i|$Heavy outliers (Robust fit).
03 · Deterministic

Weighted Least Squares

\[ \mathcal{L}(\theta) = \sum \left(\frac{r_i}{\sigma_i}\right)^2 \]

Used when some sensors are more reliable than others or when variance increases with the response variable.

03 · Deterministic

Constraints & Regularization

\[ \mathcal{L}(\theta) = \text{SSE} + \lambda \|\theta\|^2 \]
  • Constraints: $\theta \ge 0$ (Physical bounds).
  • Regularization: Prevents overfitting when data is limited.
03 · Deterministic

Identifiability

Can the data actually pin down the values of $\theta$?

Failures

Correlated parameters where one compensates for the other.

Remedies

Richer data, fewer parameters, or prior constraints.

Part III

Optimisation

04 · Optimisation

Calibration as Optimisation

\[ \hat{\theta} = \arg\min_{\theta \in \Theta} \; \mathcal{L}(\theta) \]

Key choices: Optimiser type (Gradient vs Grid), Initial Guess ($\theta_0$), and Stopping Criteria.

04 · Optimisation

Gradient Descent

\[ \theta_{k+1} = \theta_k - \eta \nabla \mathcal{L}(\theta_k) \]
  • Step size $\eta$ is critical.
  • Adaptive methods like Adam are faster in practice.
04 · Optimisation

Local Minima & Multi-Start

Non-convex losses have multiple valleys. A single run might get stuck in a "bad" local minimum.

Strategy: Run the optimizer from many random initializations and keep the best result.
04 · Optimisation

Levenberg–Marquardt

The standard for Non-linear Least Squares. It blends Gradient Descent (when far from target) and Gauss-Newton (when near target).

04 · Optimisation

Scaling and Units

If $\theta_1$ is $10^6$ and $\theta_2$ is $10^{-6}$, solvers struggle. Reparameterize or normalize your variables before solving.

Part IV

Diagnostics & Validation

05 · Validation

Residual Diagnostic Patterns

  • Curved Trend: Missing non-linearity in model.
  • Funnel Shape: Variance increases with x.
  • Autocorrelation: Missing temporal dynamics.
05 · Validation

Handling Outliers

Huber Loss

MSE near zero, MAE further out.

Trimming

Removing extreme noise (requires justification).

Reporting

Never hide outliers; explain them.

05 · Validation

Train / Val / Test Splits

Avoid memorizing noise. 1. Train: Fit parameters. 2. Val: Select model. 3. Test: Final evaluation.

05 · Validation

Cross-Validation

\[ \text{CV Error} = \frac{1}{K}\sum \mathcal{L}(\hat{\theta}_{-k};\, D_k) \]

Essential when data is limited. Divide into K folds and rotate which one is the "test" set.

05 · Validation

Walk-Forward Validation

For time series: Never shuffle. Always train on the past to predict the future.

Part V

Uncertainty Quantification

06 · Uncertainty

How sure are we?

A single point estimate is incomplete. We need:

  • Confidence Intervals: Range for $\theta$.
  • Prediction Intervals: Range for future $y$.
  • Sensitivity Analysis: What happens if $\theta$ changes 10%?
06 · Uncertainty

The Bootstrap Method

  1. Resample data with replacement.
  2. Recalibrate on the new sample.
  3. Repeat 1000x to build a distribution.
  4. Extract percentiles (2.5% and 97.5%) for 95% CI.
06 · Uncertainty

Bayesian Calibration

\[ P(\theta | D) \propto P(D | \theta) P(\theta) \]

Treats parameters as probability distributions rather than single numbers.

06 · Uncertainty

Model Discrepancy

When even the "best" $\theta$ fails, the model is likely missing a mechanism. Segment the data or add an explicit bias term.

Part VI

Probability Calibration

07 · Probability

Calibration Curves

\[ P(Y=1 | \hat{p}=q) \approx q \]

If you predict a 70% chance of rain, it should actually rain 70 out of 100 times you make that prediction.

07 · Probability

Platt & Temperature Scaling

Methods to post-process overconfident model scores into well-calibrated probabilities.

Part VII

Case Studies

08 · Case Study

Hooke's Law

\[ y = k F \]

Calibrating spring compliance $k$. Use residuals to check if the linear assumption holds at high forces.

08 · Case Study

SIR Epidemic Model

\[ \dot{I} = \beta SI - \gamma I \]

Target: Transmission rate $\beta$. Challenge: Data is under-reported. Solution: Document bias and report CI.

08 · Case Study

Team Exercise: Logistic Growth

Data provided: Population over time. Task: Find carrying capacity $K$ and growth rate $r$. Use 20% holdout for validation.
